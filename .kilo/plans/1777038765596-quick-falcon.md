# Implementation Plan - HealthScience.network v2 (The Hub)

This plan outlines the transition from a Nuxt-based website to a native HTML5/CSS3/Vanilla JS "Three-Lens Interface" featuring the Tri-Axis Hub.

## Phase 1: Cleanup and Scaffolding
- **Cleanup**: Remove Nuxt-specific files while preserving `package.json` for release tracking.
    - Remove: `.nuxt/`, `.output/`, `app.vue`, `assets/`, `components/`, `dist/`, `nuxt.config.js`, `tailwind.config.js`.
    - Retain: `package.json`, `.git/`, `.kilo/`, `js/heli-clock.js`, `bblogo.png`.
- **Directory Structure**:
    - `index.html`: Main entry point.
    - `css/`: Global and component-specific styles.
    - `js/`: Application logic and state management.
    - `js/components/`: Web Component definitions.
- **Package.json Update**: 
    - Remove Nuxt/Vue dependencies and devDependencies.
    - Update scripts to use a simple dev server (e.g., `sirv` or `http-server`).
    - Bump version to `2.0.0-alpha`.
    - Keep "name" and "version" tracking logic as requested.

## Phase 2: Core Architecture (The Hub)
- **State Management**:
    - Implement a vanilla `StateController` to manage the active `lens` state (`emulation`, `poetry`, `skeleton`).
    - Sync state with URL fragments (`#emulation`, etc.).
    - Ensure robust permalink support: The `StateController` must handle hash-based routing for top-level lenses and support deep-linking for autonomous web components within those lenses (e.g., `#skeleton/playground`).
- **Tri-Axis Hub Component**:
    - Create `<hs-hub-navigator>` custom element.
    - Implement the central geometric navigation (three radiating segments).
    - Use CSS Grid/Flex and absolute positioning for the hub.
- **Animations & Transitions**:
    - Define CSS Variables for the "biological pulse" rate.
    - Implement "warp" transitions (transforms and opacity) when switching lenses.

## Phase 3: Lens Implementation (Initial Tabs)
- **Emulation Lens (`<hs-lens-emulation>`)**:
    - Scaffold a high-res dark-mode canvas.
    - Set up the environment for Orgos and Gelles.
- **Poetry Lens (`<hs-poetry-bridge>`)**:
    - Focus on typography and organic layout.
    - Placeholder for the Gaia Manifesto.
- **Skeleton Lens (`<hs-skeleton-playground>`)**:
    - Implement the monospaced "terminal" container.
    - Placeholder for HOPquery input and SafeFlow-ECS state visualization.

## Phase 4: Refinement and Content Strategy
- Ensure the "Skeleton" view makes the "Emulation" code visible/inspectable (Transparency).
- Implement the "Swarm" button logic (placeholder for agent interaction).
- Finalize the CSS variable-driven "Resonance" across all views.

## Verification Plan
- **Manual Testing**:
    - Verify navigation between `#emulation`, `#poetry`, and `#skeleton`.
    - Verify responsive behavior of the central hub.
    - Ensure smooth transitions between views.
- **Code Quality**:
    - Verify that no Vue/Nuxt/TypeScript remnants remain.
    - Check for style encapsulation in Custom Elements.
